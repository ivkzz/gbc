# План проекта: Интеграция RetailCRM, Supabase и Дашборд на Next.js

Бизнес-цель: Построить систему мониторинга заказов, создающую мок-данные в RetailCRM, выгружающую их в Supabase, уведомляющую о крупных заказах (>50к ₸) в Telegram и отображающую статистику через дашборд.

## Архитектура системы

### 1. База Данных (Supabase + PostgreSQL)
- **Таблица `orders`**: будет хранить синхронизированные заказы (ID, имя клиента, телефон, статус, сумма, дата создания).
- **Авторизация**: Встроенный Supabase Auth (Email/Password).
- **RLS (Row Level Security)**: базовые политики, разрешающие чтение заказов только авторизованным пользователям.

### 2. Backend (Python + FastAPI)
Будет находиться в папке `backend`.
- **Стек**: Python 3.10+, FastAPI, Uvicorn, `retailcrm` (официальный клиент), `supabase`, `python-telegram-bot` или простой `requests` для TG API.
- **Скрипт 1 (`upload_mock.py`)**: Загрузка данных из локального `mock_orders.json` в RetailCRM (для тестирования).
- **Скрипт 2 (`fetch_orders.py`)**: Пакетная выгрузка заказов из API RetailCRM и сохранение их в таблицу `orders` в Supabase (историческая/ручная синхронизация).
- **Webhooks Endpoint (`/webhooks/retailcrm`)**: FastAPI сервер, принимающий POST вебхуки от RetailCRM (webhook на `order_create` и `order_update`).
  - Логика: При получении заказа — записываем в Supabase (upsert).
  - Уведомление: Если `total > 50 000` ₸ — делаем HTTP запрос к Telegram Bot API, отправляем сообщение в чат/группу.

### 3. Frontend (Next.js + Tailwind + shadcn/ui)
Будет находиться в папке `frontend` (уже инициализирована). Развертывание на Vercel.
- **Стек**: Next.js (App Router), Tailwind CSS, shadcn/ui.
- **Авторизация**: Защищенные роуты middleware Next.js + `@supabase/ssr` для работы сессий. Страница логина.
- **Дашборд**: Главная страница (защищенная) с визуализацией потока.
- **Графики**: Используется `Recharts` для отображения графика заказов (например, заказы по дням/месяцам, общая сумма).
- Дополнительно: Таблица последних заказов, показывающая, сработал ли хук.
